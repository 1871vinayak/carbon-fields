Containers

Container creation and usage

What is a container?

Carbon Container is a group of custom fields and display options. Containers are displayed on different parts of the backend, accoding to their type and display options.

Containers have a title, which must unique for every container you create. The title should reflect the purpose of the container.

Example

Carbon_Container::factory('custom_fields', 'Custom Data')
	->show_on_post_type('page')
	->add_fields(array(
		Carbon_Field::factory('map', 'location')->set_position(37.423156, -122.084917, 14),
		Carbon_Field::factory('choose_sidebar', 'custom_sidebar'),
		Carbon_Field::factory('image', 'Photo'),
	));

Container creation

To create a new Carbon Container, you should use the container factory Carbon_Container::factory($type, $title), where $type is identificator of the container type (currently accepted values are custom_fields or theme_options) and $title is the title of the container.

Types

Custom fields

What is a custom field container?

Custom field containers are used to extend the post edit screens with additional fields. Field data is stored separately for each post as post meta (see add_post_meta).

Example
Carbon_Container::factory('custom_fields', 'Post Properties')
	->add_fields(array(
		Carbon_Field::factory('map', 'location')->set_position(37.423156, -122.084917, 14),
		Carbon_Field::factory('choose_sidebar', 'custom_sidebar'),
	));
Visibility options

Custom fields containers are very flexible in terms of display options. You can select specific post type they show on, as well as category, format, parent, etc. A list of all options is displayed below:

post type

Carbon_Container::factory(...)->show_on_post_type('page')

You can also show a single container on multiple post types, as seen below:

Carbon_Container::factory(...)->show_on_post_type(array('page', 'my_custom_post_type', 'post'))

categories and custom taxonomies

Containers may be assigned to posts from specific categories or taxonomies:

Carbon_Container::factory(...)->show_on_category($category_slug)

Carbon_Container::factory(...)->show_on_taxonomy_term($term_slug, $taxonomy)

pages and subpages

Show container on a specific page, identified by path:

Carbon_Container::factory(...)->show_on_page($page_path)

Show container on all subpages of a specific page, identified by path:

Carbon_Container::factory(...)->show_on_page_children($parent_page_path)

page templates

Containers may be assigned to pages using specific template:

Carbon_Container::factory(...)->show_on_template($template_path)

The $template_path is the name of the template file, for example: "about_us.php"

post formats
To display a container posts with specific format, use:

Carbon_Container::factory(...)->show_on_post_format($post_format)

Accessing field values

To access field values you need to use the function carbon_get_post_meta($id, $name = null, $type = null), where:

$id (optional)
Post ID where your value was entered. Defaults to current post ID.
$name
The name of the field to be retrieved.
$type (optional)
If the field you want to retrieve is of type complex, you need to pass "complex".
Example
<p>Article was published in: <?php echo carbon_get_post_meta('location'); ?></p>
<p>Photo taken by: <?php echo carbon_get_post_meta(132, 'photo_author'); ?></p>

Theme Options

What is a theme option container?

Theme option containers are used to add pages with options in the back-end. Field data is stored as options.

By default, theme options containers automatically create main page in the admin area menu named "Theme Options" and sub page named "General". In most cases these default settings are sufficient, but if you need to change the title or the location of a page in the menu, read the "Multiple option pages" section below.

Example
Carbon_Container::factory('theme_options', 'General')
	->add_fields(array(
		Carbon_Field::factory('text', 'facebook_url'),
		Carbon_Field::factory('textarea', 'footer_text')
	));
Multiple option pages

It is sometimes needed to create more than one option page, at other times you need to place different pages in different sections in the admin menu. For example, you might have extensive list of settings for the background that you would want to place on a separate Theme options page under Appearance.

Below you see sample code for creating two theme option containers:

Example
// Default options page under "Theme Options"
Carbon_Container::factory('theme_options', 'General')
	->add_fields(array(
		Carbon_Field::factory('header_scripts', 'header_script'),
		Carbon_Field::factory('footer_scripts', 'footer_script'),
	));

// Add second options page under "Appearance"
Carbon_Container::factory('theme_options', 'Customize Background')
	->set_page_parent('themes.php')	// name of the "Appearance" admin section
	->set_page_file('customize-background.php')
	->add_fields(array(
		Carbon_Field::factory('color', 'background_color'),
		Carbon_Field::factory('image', 'background_image')
	));
Page options

Each theme options container is identified by it's title, page file and page parent.

Title
This is the title you give to the container. It is displayed in the admin menu, as well as on top of the page itself.
Page file
This is a name that WordPress will use to uniquely identify your page in the admin area. It should be URL friendly (lowercase, hyphen-separated) and end with ".php". This corresponds to the $menu_slug parameter of add_submenu_page.
Page parent
This is the identificator of a top level menu section in the admin menu sidebar. This corresponds to the $parent_slug parameter of add_submenu_page. You can see all predefined page parents here.

For detailed information about managing admin pages, see Administration_Menus.

Permissions

Every theme options container requires a level of permission, which by default is set to edit_themes (read more about permissions: Roles & Capabilities).
You can change the permission required to view your options page using set_page_permissions($permission)

Accessing field values

To retrieve field values from a theme options container, you need to use the function carbon_get_theme_option($name, $type = null), where:

$name
The name of the field to be retrieved.
$type (optional)
If the field you want to retrieve is of type complex, you need to pass "complex".
Example
<p>Copyright <?php echo carbon_get_theme_option('copyright'); ?></p>
<p>
	Office locations:
	<?php 
	$address_lines = carbon_get_theme_option('addresses', 'complex');
	foreach ($address_lines as $line) {
		echo $line . '<br/>';
	}
	?>
<p>
